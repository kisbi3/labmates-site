01 · Getting started

새 프로젝트를 가져오는 대신, 이미 있는 폴더를 엽니다.

Asterelle은 개인 연구자의 로컬 또는 SSH 워크스페이스를 엽니다. Markdown, 코드, 데이터와 Git 저장소는 기존 위치에 남으며 전용 데이터베이스로 강제 이전되지 않습니다.

  1. 01 / OPEN연구 폴더 또는 Asterelle 설치 워크스페이스를 엽니다.
  2. 02 / DISCOVER프로젝트 경계, 문서 링크, Git과 작업 상태를 읽습니다.
  3. 03 / CONTINUEObservatory에서 최근 변화와 다음 판단을 확인합니다.

현재 필요한 것

  • macOS, Windows 또는 Linux 데스크톱 환경
  • 연구 파일을 담은 로컬 또는 접근 가능한 SSH 워크스페이스
  • AI 작업을 사용할 경우 기존 Claude Code, Codex 또는 설정된 provider 접근 권한

02 · Core concepts

파일보다 넓고, 연구자보다 앞서지 않는 상태 모델.

Workspace

연구자가 연 설치 또는 폴더의 경계입니다. 파일, Git, 터미널과 파생 인덱스가 이 범위 안에서 작동합니다.

Project

목표, 작업 보드, Mates와 근거가 귀속되는 연구 단위입니다. 여러 프로젝트를 한 워크스페이스에서 볼 수 있습니다.

Candidate

에이전트나 자동화가 준비했지만 아직 정본으로 승인되지 않은 결과입니다.

Canonical record

연구자가 신뢰하고 승인한 프로젝트 기록입니다. Markdown과 프로젝트 파일이 중심이며 인덱스는 재생성할 수 있습니다.

03 · Observatory

모든 프로젝트에서 지금 판단할 일을 모읍니다.

Observatory는 일반적인 활동 피드가 아닙니다. 프로젝트별 최근 변화, 결정 대기 항목, 위험 신호, 야간 결과와 이어서 할 작업을 근거와 함께 보여주는 개인 연구 운영 화면입니다.

  • Lab signals: 프로젝트를 넘나드는 중요한 변화와 연결 후보
  • Continue working: 현재 세션의 활성 프로젝트와 열린 문서
  • Projects at a glance: 모든 프로젝트의 상태, 변화와 다음 행동
  • Needs your attention: 검토·승인·기각해야 하는 후보

프로젝트 필터는 전역 화면을 교체하지 않고 관련 프로젝트와 결정 큐만 좁힙니다.

04 · Mates and models

모델은 바뀌어도 프로젝트 맥락은 이어집니다.

Mates는 단순한 채팅 캐릭터가 아니라 프로젝트, 역할, provider, 모델과 실행 기록을 가진 연구 작업자입니다. Asterelle은 구성에 따라 Claude Code, Codex, 로컬 모델과 Hermes provider를 역할별로 사용할 수 있습니다.

  • 조율, 코딩, 독해, 요약, 야간 준비와 검증에 서로 다른 모델 배치
  • 작업과 모델이 바뀌어도 같은 프로젝트 범위와 근거 참조 유지
  • 로컬·원격 여부와 fallback 결과를 실행 기록에 표시
  • 작성 역할과 독립 검증 역할을 분리할 수 있는 구조

가용 모델과 외부로 전달되는 정보는 사용자가 설정한 provider 및 워크플로 문맥에 따라 달라집니다.

05 · Evidence and ratification

에이전트의 결과는 답변이 아니라 검토할 후보입니다.

중요한 후보는 제안 내용, 필요한 이유, 실행 기록, 파일·Git 근거, 작성자와 검증 상태를 함께 제공합니다. 연구자는 적용될 변경을 확인한 뒤 승인, 기각 또는 보류합니다.

  1. 에이전트 또는 자동화가 제한된 범위에서 후보를 준비합니다.
  2. 필요한 경우 작성자와 다른 역할이 결과를 검증합니다.
  3. 연구자가 근거와 변경 미리보기를 확인합니다.
  4. 승인된 후보만 정본 연구 상태로 승격됩니다.

에이전트는 자신의 결과에 필요한 사람의 확인을 스스로 발급할 수 없습니다.

06 · Data, privacy, and recovery

원본은 읽을 수 있는 파일로 남고, 파생 상태는 다시 만들 수 있습니다.

  • Markdown과 Git 저장소는 사용자의 워크스페이스에 남습니다.
  • 그래프와 검색 인덱스는 원본이 아니라 재생성 가능한 파생 상태입니다.
  • 시스템 Python을 수정하지 않고 제품 전용 위치에 런타임을 둡니다.
  • Night Automation은 기본적으로 선택 사항이며 설정된 작업과 한도를 따릅니다.
  • 원격 모델 호출 범위는 해당 provider와 워크플로 설정을 확인해야 합니다.

문제가 생겼을 때
